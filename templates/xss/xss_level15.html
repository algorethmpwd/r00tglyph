Level 15: XSS via Template Injection

Challenge Description

Welcome to TemplateX, a fictional email template builder! This challenge simulates a real-world scenario where a template injection vulnerability can be exploited to achieve XSS.

Mission Briefing

Client: TemplateX Inc.
Target: Email Template Builder
Vulnerability: Server-Side Template Injection
Objective: Exploit template injection to execute JavaScript

Technical Background: Template injection occurs when user input is embedded in a template that is then processed by a template engine. If the input is not properly sanitized, attackers can inject template syntax that will be executed by the engine. This can lead to various attacks, including XSS, server-side code execution, and more.

Real-world Impact: Template injection vulnerabilities are particularly dangerous because:

  • They can lead to both client-side and server-side attacks
  • They often allow access to the application's internal objects and methods
  • They can bypass traditional XSS protections
  • They can lead to remote code execution in some cases
  • They're common in applications that use template engines like Handlebars, Mustache, Jinja2, etc.

Your Task: In this challenge, you need to exploit a template injection vulnerability in the email template builder. The application uses a simple template engine that allows variables to be embedded in templates. You need to find a way to inject template syntax that will execute JavaScript. Make an alert box appear with the text "XSS Level 15 Completed!" to reveal the flag.

Hint: The template engine uses double curly braces {{ "{{ }}" }} for variables and supports some basic expressions and functions. Try to find a way to execute JavaScript through the template syntax.
