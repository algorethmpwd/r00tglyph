Level 6: XSS with ModSecurity WAF

Challenge Description

Welcome to SecureBank, a fictional banking platform! You've been hired to test their customer support portal which is protected by a ModSecurity-like Web Application Firewall (WAF).

Mission Briefing

Client: SecureBank Financial Services
Target: Customer Support Portal
Protection: ModSecurity WAF Rules
Objective: Bypass the WAF and execute JavaScript in the browser

Technical Background: ModSecurity is a popular open-source Web Application Firewall (WAF) that provides protection against various attacks, including XSS. It uses sophisticated rule sets to detect and block malicious payloads. This level simulates a ModSecurity implementation with OWASP Core Rule Set (CRS).

Real-world Impact: In real-world scenarios, bypassing a WAF could allow attackers to:

  • Execute sophisticated attacks that evade security monitoring
  • Compromise high-security environments that rely on WAF protection
  • Exploit vulnerabilities in critical infrastructure
  • Access sensitive financial or personal information
  • Perform account takeovers in banking applications

Your Task: Bypass the ModSecurity WAF and make an alert box appear with the text "XSS Level 6 Completed!" to prove the vulnerability exists and reveal the flag.

Hint: ModSecurity uses pattern matching to detect attacks. Can you find a way to encode or obfuscate your payload to evade detection?
