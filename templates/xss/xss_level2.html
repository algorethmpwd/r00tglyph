Level 2: DOM-based XSS

Challenge Description

Welcome to ColorPicker Pro, a fictional web application for designers! You've been hired to test the security of their color preview feature.

Mission Briefing

Client: ColorPicker Pro
Target: Color Preview Feature
Vulnerability: Suspected DOM-based Cross-Site Scripting (XSS)
Objective: Confirm the vulnerability by executing JavaScript in the browser

Technical Background: DOM-based XSS occurs when JavaScript code takes data from a user-controllable source (like URL parameters) and passes it to a sink that supports dynamic code execution. Unlike reflected or stored XSS, the vulnerability exists entirely in client-side code.

Real-world Impact: In a real scenario, this vulnerability could allow attackers to:

  • Execute arbitrary JavaScript in the victim's browser
  • Access sensitive information from other parts of the page
  • Bypass client-side security controls
  • Perform actions that appear to come from the victim

Your Task: Make an alert box appear with the text "XSS Level 2 Completed!" to prove the vulnerability exists and reveal the flag.

Hint: Look at how the color parameter is used in the URL and how it affects the page. What happens if you provide something other than a valid color?
