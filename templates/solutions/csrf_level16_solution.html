Level 16 Solution: CSRF with CSP Bypass

Challenge Overview

This challenge demonstrates advanced CSRF attacks that bypass Content Security Policy (CSP) protections through various techniques including JSONP, AngularJS injection, and meta tag manipulation.

Tools Required

  • Burp Suite: For CSP header analysis and request manipulation
  • Browser Developer Tools: To inspect CSP violations and responses
  • CSP Evaluator: Online tools to analyze CSP policies
  • Knowledge of CSP directives: Understanding script-src, form-action, base-uri

Step-by-Step Solution

  1. Analyze CSP Configuration: Select a CSP header with misconfigurations
  2. Choose Payload Method: Select appropriate delivery method (form, fetch, iframe, meta)
  3. Enter Bypass Technique: Use one of these techniques:
    • jsonp - For JSONP callback exploitation
    • angular - For AngularJS template injection
    • meta - For meta refresh attacks
    • base - For base URI manipulation
    • iframe - For iframe-based attacks
  4. Execute Attack: Submit the form to test the bypass
Key Insight: CSP bypass techniques exploit misconfigurations in Content Security Policy directives.

Security Impact

This CSRF vulnerability demonstrates how attackers can:

  • Execute unauthorized actions on behalf of authenticated users
  • Bypass authentication mechanisms through cross-site requests
  • Potentially compromise user accounts and sensitive data
  • Perform state-changing operations without user consent

Mitigation Strategies

  • CSRF Tokens: Implement anti-CSRF tokens in all state-changing forms
  • SameSite Cookies: Use SameSite cookie attributes to prevent cross-site requests
  • Origin Validation: Verify the Origin and Referer headers
  • Custom Headers: Require custom headers for AJAX requests
  • Double Submit Cookies: Implement double submit cookie pattern
