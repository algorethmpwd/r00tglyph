Command Injection Level 14 - Challenge Information
Objective:

Exploit command injection through WebSocket messages in a real-time monitoring system.

Scenario:

You're testing a modern infrastructure monitoring platform that uses WebSockets for real-time communication. The system processes JSON messages containing monitoring commands and executes them to provide live system information.

Learning Goals:
  • Understand command injection in WebSocket communications
  • Learn about real-time application vulnerabilities
  • Practice JSON-based payload construction for WebSockets
  • Explore modern real-time system security
Modern Technology Context:

This challenge represents vulnerabilities in:

  • Real-time monitoring dashboards
  • Live chat applications
  • Gaming platforms with real-time features
  • Collaborative tools (like Google Docs, Slack)
  • IoT device management platforms
  • Financial trading platforms
WebSocket Security Considerations:
  • WebSocket messages bypass traditional HTTP security controls
  • Real-time nature makes detection more difficult
  • JSON message processing often lacks proper validation
  • Persistent connections can be exploited for sustained attacks
Hints:
  • Inject commands within JSON message values
  • Try system monitoring commands like 'ps' or 'netstat'
  • Use command separators within JSON string values
  • Test different JSON fields for injection opportunities
