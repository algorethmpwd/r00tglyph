Challenge Description: SQL Injection with UNION

Challenge Description

Welcome to LibrarySystem, a fictional library management system! This challenge simulates a real-world scenario where a book search feature is vulnerable to UNION-based SQL injection.

The developers of LibrarySystem have implemented a search feature that allows users to find books by title. However, they've made a critical mistake in how they handle user input in the search query, allowing attackers to use UNION statements to extract data from other tables in the database.

Your Task: Exploit the SQL injection vulnerability in the search feature to extract data from the hidden users table. The administrator's username is admin, and their password contains the flag. Use a UNION-based SQL injection attack to reveal this information.

Hints

The search feature likely uses a SQL query similar to: 

SELECT id, title, author, category, year FROM books WHERE title LIKE '%search_term%'

To use a UNION attack, you need to:

  1. Determine the number of columns in the original query
  2. Ensure the data types in your UNION query match the original query
  3. Use the UNION to query the users table instead of the books table
